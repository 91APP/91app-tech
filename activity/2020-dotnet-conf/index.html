    91APP 企業實戰系列

    DevOps 教戰手冊：三步工作法

    第一場由我們的敏捷教練 Ruddy 老師，為大家帶來 DevOps 三步工作法的介紹。91APP 的服務，都是以 Software As A Service 的模式，提供給我們的客戶使用。產品研發團隊同時肩負了開發 (Dev) 與維運 (Ops) 的任務。Ruddy 老師是我們的敏捷教練，帶給我們的 DevOps 知識，是我們能夠妥善的開發與維運我們自身服務的重要關鍵。

    非同步系統的服務水準保證；淺談非同步系統的 SLO 設計

    第二場由 91APP 架構師 Andrew 為大家說明，DevOps 團隊該如何靠良好定義的 SLO 來維持服務的水準。在高流量的場景下，處理任務除了要「快」之外，針對特定任務還需要同時注重「精確」才行。這背後包含了技術、開發與管理的考量與決策。我們用認證簡訊的發送來當作案例，說明我們如何靠 SLO 的定義，來完成 Dev 與 Ops 的改善循環。

    微型任務編排器 - 以 Process Pool 為例

    第三場，則由 91APP 架構團隊的資深工程師 Steven 為大家說明，在我們每天處理上千萬個任務背後的系統，如何做好工作排程來達到最佳的效率？這個主題延續第二場的內容，除了良好的服務水準定義 (SLO) 之外，也說明了背後的工作排程器 (Process Pool) 如何設計達成目標。對於想要精通 .NET Core / K8S 的技術人，這是場一定要聽的實際案例分享。

    刻意練習：如何鍛鍊你的抽象化能力

    最後一場的主題，我們請到了架構團隊的架構師 Andrew Wu 與工程師 Fion 來現身說法，聊聊 .NET 開發團隊精通 OOP 觀念的重要性，以及 91APP 架構團隊如何讓成員透過有計畫的練習，來提升 C# / OOP 觀念與架構設計能力的做法。物件導向強大的威力，建構在你的程式碼是否精準的描述你在現實世界要處理的問題。開發出來的程式碼品質，有70% 是取決於你的程式碼架構（也就是 class 與 interface 的設計）是否精準，剩下的才是背後的實作。這些能力對於架構團隊是很關鍵的技能，如果你想要提升 OOP 的設計能力，請千萬不要錯過這個議程。
